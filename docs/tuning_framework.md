# Фреймворк доводки прошивки под реальный вертолёт

Принцип: сначала ИЗМЕРИТЬ объект, потом крутить регуляторы. Каждое число в
прошивке должно опираться на замер или на явно записанное решение пилота.

## 1. Идентификация железа (стенд, разовые замеры)

| Что меряем | Как | Зачем | Статус |
|---|---|---|---|
| Задержка хвостового ESC+мотора+винта | степ-тест: ступенька на ESC, гиро-Z пишется в RAM-лог (g_step_*) | задаёт ПОТОЛОК гейнов хвоста; решение «менять ли ESC» | **ЗАМЕРЕНО 13.07: вверх мёртвое ~20-30мс + выход ~60мс = ~80-90мс; предсказание по частоте качелей (83мс) подтвердилось — KP=2 честный потолок** |
| Асимметрия вверх/вниз | тот же тест, ступенька вверх и вниз | калибрует NEG_BOOST честно | **вниз мёртвое ~60мс (нет активного торможения, винт тормозится сам) — вдвое медленнее верха** |
| Спектр вибрации по оборотам | гиро-лог на разных оборотах, FFT на хосте | выбор частот DLPF/LPF не вслепую | позже |
| Обороты ротора по газу | фундаментал вибрации в гиро-FFT | кривая газа, головная скорость | позже |

Критерий решения по ESC: суммарная задержка (мёртвое время + τ до 63%)
**> ~40 мс → заказывать современный микро-ESC** (BLHeli_S 6-12A: отклик ~5-15 мс,
опционально bidirectional = реверс-фича). Сегодняшние качели 3-4 Гц предсказывают
~80 мс — тест подтвердит или опровергнет.

## 2. Что стенд меряет честно, а что нет

Честно: знаки/направления, feedforward-структуры, фильтры, задержки железа,
арминг/фейлсейф, кривая газа.
НЕчестно: финальные гейны замкнутых петель. Стоя на столе rate-петля циклики не
замыкается (рама не поворачивается за тарелкой), хвост колеблется через
трение-прилипание о стол. На столе ловим только грубую устойчивость.

## 3. Протокол висения (когда дойдём)

Порядок жёсткий, один винт за вылет, после каждого — запись в протокол:
1. Хвост: жёсткость удержания вверх шагами (KI, потом KP) до первого виляния,
   откат на шаг. База анти-момента: подобрать FF так, чтобы PID-коррекция в
   висении была ~0 (читается по SWD после посадки из лога).
2. Циклика rate: KP вверх до звона, откат; KI — по возврату в курс после порыва.
3. Ощущения пилота: резкость стика (CYCLIC_GAIN / STICK_TO_DPS), кривая газа.

## 4. Инструментарий

- mon_rate.tcl — здоровье радио (loss/resync/maxPollGap).
- Степ-тест ESC: `mww <g_step_go> 1` при газе в нуле; лог 256×10 мс в RAM,
  читать после прогона (адреса — nm firmware.elf | grep g_step).
- Все SWD-замеры при работающем главном моторе — только малые обороты
  (наводка роняет ST-Link), либо RAM-лог с чтением после.
- Свежая силовая батарея перед каждой сессией (просадка = ребут ESC).

## 5. Ворота аппаратных решений

| Решение | Критерий | Статус |
|---|---|---|
| Новый хвостовой ESC | задержка >40 мс по степ-тесту | **ЗАКАЗЫВАТЬ: замерено 80-100мс.** Брать микро BLHeli_S 6-12A с damped light (активное торможение лечит медленный сброс) + возможность bidirectional (открывает реверс-фичу) |
| Реверс хвоста | только с новым bidirectional ESC | отложено (эскиз в памяти), разблокируется покупкой выше |
| Мягкий подвес IMU | если вибрация в гиро мешает и после фильтров | наблюдаем |
